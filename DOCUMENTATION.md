## Documentation Principles

Our documentation follows these principles:

1. **User-First:** Written for users, not developers
2. **Example-Driven:** Show, don't just tell
3. **Progressive:** Start simple, add complexity
4. **Searchable:** Good headings and structure
5. **Up-to-Date:** Kept in sync with code
6. **Comprehensive:** Cover common and edge cases
7. **Honest:** Acknowledge limitations
